# Структура директории .specify

Эта директория содержит все спецификации проекта, документацию и организационную память для проекта universo-platformo-next.

## Структура директорий

```
.specify/
├── memory/           # Память проекта и база знаний
│   ├── constitution.md      # Основные принципы, стандарты и руководства
│   ├── analysis/            # Архитектурный анализ и обзоры
│   └── archive/             # Архивированные исторические документы
├── specs/            # Спецификации функций
│   └── NNN-feature-name/    # Отдельные спецификации функций
│       ├── spec.md          # Спецификация функции
│       ├── tasks.md         # Задачи реализации
│       ├── plan.md          # План реализации
│       ├── data-model.md    # Модели данных и схемы
│       ├── research.md      # Исследования и решения
│       ├── quickstart.md    # Краткое руководство
│       ├── checklists/      # Чеклисты качества
│       └── contracts/       # API-контракты
├── scripts/          # Утилиты и скрипты
│   └── bash/                # Bash-скрипты для автоматизации
└── templates/        # Шаблоны документов
    ├── spec-template.md        # Шаблон спецификации функции
    ├── tasks-template.md       # Шаблон документа задач
    ├── plan-template.md        # Шаблон плана реализации
    ├── checklist-template.md   # Шаблон чеклиста
    └── agent-file-template.md  # Шаблон конфигурации агента
```

## Описание директорий

### `/memory`
Содержит институциональные знания проекта:
- **constitution.md**: Авторитетный источник принципов проекта, технологического стека, архитектурных паттернов и стандартов разработки
- **analysis/**: Архитектурные анализы, документы лучших практик и результаты обзоров
- **archive/**: Исторические документы, которые больше не используются активно, но сохранены для справки

### `/specs`
Спецификации функций, организованные по пронумерованным директориям (NNN-feature-name):
- Каждая функция получает свою собственную директорию с трехзначным префиксом для упорядочивания
- Содержит всю документацию для конкретной функции или эпика
- Следует шаблонам из директории `/templates`

### `/scripts`
Скрипты автоматизации для операций проекта:
- **bash/**: Shell-скрипты для проверки предварительных условий, валидации и автоматизации

### `/templates`
Стандартные шаблоны для создания новой документации:
- Используются инструментами автоматизации и разработчиками
- Обеспечивают согласованность во всех спецификациях

## Руководство по использованию

### Создание новой спецификации функции

1. Создайте новую директорию: `.specify/specs/NNN-feature-name/`
2. Скопируйте шаблоны из `.specify/templates/`
3. Заполните спецификацию, следуя структуре шаблона
4. Обратитесь к `.specify/memory/constitution.md` для соответствия стандартам

### Обновление конституции

Конституция является источником истины для стандартов проекта. Обновления должны:
1. Быть предложены с четким обоснованием
2. Следовать процессу внесения поправок, определенному в конституции
3. Включать увеличение версии (семантическое версионирование)
4. Документировать все изменения с анализом влияния

### Добавление аналитических документов

Новые архитектурные анализы, обзоры или резюме должны быть размещены в:
- `.specify/memory/analysis/` - Для активных справочных документов
- `.specify/memory/archive/` - Для исторических документов

## Интеграция с .github/agents

Директория `.specify` доступна агентам GitHub Copilot в определенных режимах:
- **tasks** режим: Может обращаться к `.specify/` для генерации задач и планирования
- **analytics** режим: Может обращаться к `.specify/` для анализа проекта

Эта интеграция настроена в конфигурационных файлах `.github/agents/`.

## Стандарты форматирования документов

Все документы в `.specify/` должны следовать этим стандартам:

1. **Markdown**: Используйте стандартный GitHub-flavored Markdown
2. **Двуязычность**: Предоставляйте версии на английском и русском языках, где требуется
3. **Структура**: Точно следуйте структуре шаблона
4. **Именование**: Используйте строчные буквы с дефисами (kebab-case)
5. **Длина строки**: Предпочитайте 120 символов или менее для читаемости
6. **Блоки кода**: Всегда указывайте язык для подсветки синтаксиса
