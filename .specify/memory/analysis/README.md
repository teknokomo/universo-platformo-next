## Usage Guidelines

### Reading Order for New Contributors
1. Start with ARCHITECTURAL-ANALYSIS.md for project overview
2. Read BEST-PRACTICES-INTEGRATION-SUMMARY.md for development standards
3. Review REVIEW-FINDINGS.md for known issues and improvements
4. Check PLANNING-SUMMARY.md for project roadmap

### Document Maintenance
- Archive outdated documents to `/archive` subdirectory
- Keep bilingual documents (EN/RU) synchronized
- Update this index when adding or removing documents

### Document Naming Convention
- Use UPPERCASE-WITH-HYPHENS for analysis documents
- Add -RU suffix for Russian versions
- Use descriptive names that clearly indicate content
